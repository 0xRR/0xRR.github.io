Lychee--荔枝图床-docker-部署-记录

Posted on 2018-12-05 | In 3C & 数码 |

简介

Lychee
是一款开源的图片管理系统，实际上也可以用作图床，一直在更新，自身功能很强大，还支持二次开发插件及扩展来丰富功能，此外这套系统也支持
Docker 。

官网地址

项目 Github 地址

Demo 演示

Docker

部署

以ubuntu16.4为例

安装docker环境

sudo apt-get install docker.io

拉下镜像

sudo docker pull kdelfour/supervisor-docker

建立文件夹并设置权限（可跳过）

cd /root

mkdir lychee

cd lychee

mkdir data

mkdir uploads

mkdir mysql

mkdir uploads/import

mkdir uploads/big

mkdir uploads/thumb

chmod -R 777 uploads/ data/ mysql/

如果不手动建立，就不要给容器指定外部目录，否则会报错权限不足[1][2]

Unable to save this configuration. Permission denied in \’data/\’.
Please set the read, write and execute rights for others in
\’data/\’and \’uploads/\’. Take a look at the readme for more
information.

如果有其他报错，可以进入lychee后点击左上方小齿轮，点击Diagnostics查看错误

报错后，修改文件夹权限之后，可使用如下命令重启容器

查看正在运行的容器[3]：

docker ps

查看所有容器[3]：

docker ps -a

重启容器[3]：

docker start \

\

运行镜像

sudo docker run -it -d -p 80:80 -v /your-path/uploads/:/uploads/
-v /your-path/data/:/data/ -v /your-path/mysql/:/mysql/
kdelfour/lychee-docker

替换your-path为刚刚新建目录的地址，例如在/root下的话就是：

sudo docker run -it -d -p 80:80 -v /root/lychee/uploads/:/uploads/ -v
/root/lychee/data/:/data/ -v /root/lychee/mysql/:/mysql/
kdelfour/lychee-docker

如果跳过了刚刚的建立文件夹及权限设置，数据库及图片将会保存在容器内部，则使用下列命令运行镜像：

sudo docker run -it -d kdelfour/lychee-docker

\

设置自启动

在运行docker容器时可以加如下参数来保证每次docker服务重启后容器也自动重启[4]：

docker run

--restart=always

如果已经启动了则可以使用如下命令[4]：

docker update --restart=always \

\

配置

运行镜像后，浏览器中输入服务器IP以登录

首次登录需要初始化数据库，使用如下配置

user name : lychee

user password : lychee

然后就是设置管理账号与密码，设置一个足够复杂的密码就可以了

\

其他事项

设置图片公开，可以得到一个直接可访问的链接

然而，对此照片取消公开，并不会使此链接失效……

意味着，只要设为公开，发布了此链接，就永远公开了，想取消公开只能删除图片

\

不过，可以使用WinSCP、Xftp、等其他方式，替换掉
/your-path/lychee/uploads/big/
下面的对应文件，在不更换图片链接的情况下，对外链的图片进行修改

Mavic Pro-Air-大疆无人机使用心得体会

Posted on 2018-04-10 | In 3C & 数码 |

1.记得插tf卡，插电脑上考出来之后马上插回去，不然迟早得忘

2.起飞、降落的时候拿着屁股对着自己，不然的话脑子一抽心一慌打杆反了妥妥的炸

3.非恶劣天气下，高空飞行建议取下云台保护罩，减轻飞行重量，避免画质损失。关键是从高空炸鸡炸下来那货也保护不到什么。。。。（恶劣天气、低空作死飞行可以带上

4.时刻关注电量，记不住的话把低电量报警调高点，等开始报警了嫌吵的话再调低点都可以；电量条上的最低返航电量是有一些富裕的，但是为了安全起见，避免遇到意外情况时却没有足够的电量了，最好还是听从其建议。

5.关注电池电压（可在设置中打开显示到屏幕上），大幅度打杆会导致大电流放电，电压会掉很多，希望电池寿命长点、飞行续航长点的话应该避免，比如S模式全速向前全速爬升一会儿就电池过流警报了，自己看着电压打杆就不会触发放电过流警报

6.起飞前观察环境，风向，阳光方向，自己所站的朝向

风向会影响返航时的耗时，如果顺风出去，回来的时候就会耗时更久。强烈的阳光会触发前视觉避障，如果太阳很大，最好是把自动返航避障爬升关了（前提是做好飞行计划，见后一条）。自己所站的朝向、以及飞机所在的位置，要结合实际地形、指南针、地图来综合考虑，根据飞机的位置调整自己（也是遥控器天线）的朝向，以获得最好的信号。

7.起飞前做好飞行计划，想好自己往哪个方向飞，路径上有哪些阻隔，返航高度应该设为多少。

起飞之前估计设置一个返航高度，飞出去之后，尽快爬升到所有障碍物之上，然后记下高度，再根据实际需要重新设定返航高度，设高了浪费电划不来，设低了有危险。

8.天线位置是在前桨臂上，如果希望获得更远的遥控和图传范围，可以选择爬高后倒飞或侧飞，不过需要千万确保没有障碍物

9.如在计划之外，临时起意想拍摄视频、照片，一定记得确保确定自己的航线上没有障碍物，特别是向下、向后、侧向移动时的拍摄，建议先原地360转一圈看看，再向下45度转一圈看看，再向下90度看一眼。

如需要下降的时候，可以调转机头朝自己所站的位置看一眼，看下降多少会有阻隔，尽量保持在安全高度之上，虽然失控了会自动爬升，但是会干扰飞行和拍摄。

10.下降速度只有3米/秒，如果飞高了，返航的时候（在确保安全的前提下），记得全程下降打满，尽量避免已经飞到自己头顶了，却还有400米的高度需要下降。。。

11.建议自己做一个
起飞前检查清单，每次起飞的时候照着清单走一遍，费不了多少时间，但是很有帮助

12.电池充电时温度会升高，如果升高到一定程度会自动减少电流以避免损伤，所以可以用一个小风扇吹着电池给他充电。

13.电池的参数写的最大充电功率100W，而官方标配的220V充电器输出功率只有50W，应该是从慢充升温小保护电池的考虑上出发的，可以配合三方大功率充电器和小风扇一起使用

14.淘宝卖的这种防撞圈不要买，重达117g，而且安装得拆桨叶，超级麻烦，而且保护力度相当有限

15.淘宝卖的这种信号增强装置不要买，没什么暖用，有次用了之后信号还变弱了。。。而且收纳相当不方便，折叠起来比不折叠占的空间还要大。。。。

Nintendo-Switch-4G热点联机对战的一种解决方案

Posted on 2018-01-20 | In 3C & 数码 |

部分地区和运营商的手机卡（上网卡）在开手机热点或是插入随身WiFi（mifi、4G移动路由器）的时候，因为拿不到公网IP，NAT层数较多，无法联机（比如喷射战士2.。。。直连电信NAT
Type D）

除了几乎不可能的给运营商打电话投诉这一条途径，还有就是在路由器上挂SS（需要带UDP转发的SS服务器）并启动路由器上的UDP全局转发（游戏模式）这一功能（梅林、Padavan均支持）

理论上只要肯折腾，openwrt的路由器也支持，可以直接买这个，不过因为政策原因厂家也不敢把SS功能做得太好（https://www.gl-inet.com/forums/topic/shadowsocks支持/），所以需要自己弄

然而，首先我穷，600太贵了，其次懒得再去学习openwrt了，于是选择了熟悉的Padavan

首先需要一个乐携Nexx
WT3020F，淘宝售价88元，mt7620处理器16M/64M，1WAN
1LAN 1USB，5V MicroUSB供电，体积比火柴盒稍大

然后按此方法刷Padavan，全文摘抄如下：

1. 将U盘格式化为FAT32

2. 将breed.bin 和 WT3020H.trx 拷贝至U盘根目录，并连接至路由器USB接口

3. 路由器通电，稳定后，通过命令提示符Telnet 192.168.8.1
（非摘抄的注释：Win10
2017秋季更新之后，开启telnet需要从控制面板的程序和功能进入，从文件浏览器的卸载程序进不去了）

Login: nexxadmin

Password: y1n2inc.com0755

4. 进入U盘目录 cd
/media/sda/sda1（非摘抄的注释：如果多次插拔U盘，会变成sda2 sda3
sda4.。。。最好一次搞定或者重启路由器

5. 可以考虑备份原厂固件，操作命令逻辑大致如下（非必须步骤）

for i in 0 1 2 3; do cat /dev/mtdblock\$i > mtdblock\$i.bin; done

6. 刷入Breed

mtd_write -r -e mtd0 write breed.bin
mtd0（非摘抄的注释：一定要找对breed版本，不然会变砖

稍等几秒，即可完成Breed刷入

【刷入固件】

1. 保持U盘连接至路由器USB接口，通过命令提示符Telnet 192.168.8.1

刷完之后，你还需要一个USB的4G网卡，淘宝关键字为 4G 卡托
，买非WiFI版（电脑版）就行，联通3G

USB 4G网卡插入后路由器之后，在路由器设置中的 高级设置 USB应用程序

国家、运营商选你对应的，接入点联通wonet（据说比3gnet快些），移动cmnet，电信ctlte（据说比ctnet快些）

外部网络状态，如果看到获得到了IP，就说明连上4G网络了

重庆市国家税务局车辆购置税征收管理分局
重庆市九龙坡区渝州路86号，石油路彩电中心旁边

需要身份证复印件，购车发票第三联，车辆资料复印件一份

里面有个电脑，自己坐上去挨个填空，然后打印出一张报税申请表，或者在外面让打印店打印也可以

购置税的算法是，发票价格剔除增值税之后的不含税车价，乘以10%

具体算法是，比如发票是12000元，购置税就是12000/1.17*0.1=1025.641元

而不是代办上户人口中的12000*0.1=1200元，他们一般都会吃这个差价

另外，在计税系统中，有个计税最低价格，如果不含税车价（发票价格剔除17%增值税之后的价格）低于计税最低价格，就以计税最低价格计算购置税

到我这情况，官方报价32000的车，发票开的是27200，算出来不含税价格是23247，报税也就是按23247报，所以交税2324

最后报税的时候，工作人员给我说，我这车的最低计税价格是20500，最低购置税就是2050，相当于我因为发票的原因多交了274元

关于压制后的录像在索尼相册中排序错乱问题的治疗方案

出去玩，拍照，录像，回到家感觉录像占的空间太大（比特率高达18M）于是压一下，码率压到4M，再丢回手机之后，录像就在最前面了（按时间顺序排列的

试着调整创建时间、修改时间来治疗此问题，无效

最后发现，是mp4的元数据里面的编码时间决定了此视频在相册中的位置

搜了一下，果然有蛋痛之人遇到过这种问题：https://superuser.com/questions/523286/how-to-make-handbrake-preserve-capture-time-creation-time

解决方法是用FFmpeg来把原始视频文件的元数据完全的复制到压缩后的视频文件中去，真机智！

命令行是：ffmpeg -i 原文件.mp4 -i 压缩后的文件.mp4 -map 1
-map_metadata 0 -c copy正确时间的文件.mp4

技术难度和季节、气候有关，总体而言，夏末秋初应该是最好的攀登季节了，雨季过后雨水少，夜间温度又不是太低，白天出太阳无风可穿短袖。雪线之上雪层硬度、厚度适中，就我这次而言，不需要冰爪、冰镐、雪套，不用登山鞋就穿低邦徒步鞋或者越野跑鞋都可以通过选择合适的线路登顶。（夏季冰川无积雪，全是冰泥，特滑，必须要冰爪，深秋、冬天和春天冰裂缝会被新雪盖住不好判断稳固性，不像夏季的雪桥都是稳固的，而且冬天特别冷）

关于结组

我觉得登山手册里面说得很对，表达能力有限，直接上图。。。

年保结组不结组，带不带保护器材，我觉得还是取决于 季节、人员、攀登的线路

至少在9月中秋节前后，雪层厚度和硬度是非常适宜的，全队人员体能、技能相当，如果避开裂缝威胁选择合适的冲顶线路，坡度都不陡，不容易滑坠，而且即使滑坠了，也可以很安全迅速的制动，所以完全没有必要使用结组和保护器材，只会徒增负重，降低进山的行进速度、冲顶的速度，以及丧失对天气变化处理的灵活性。

举个例子，如果使用结组、保护，登顶需要4小时，下撤需要3小时，一共就是7小时，而在7小时内，天气很有可能变坏（不管上午天气如何，基本每天下午都会下点雨/雪），而要保证下午不被雨雪困扰，那就得早上早早的出发，而出发早了气温又低，云雾又没散开，增加行进难度。

而轻装不结组不设保护点冲顶，完全可以等上午10点太阳出来得大大的，一上一下下来吃午饭刚刚好。。。

总之，结组不结组，带不带保护器材，应该是一个需要事先思考的问题。

这是从小山棱右边小碎石坡爬上来，终于能看到冰川全貌了的时候，也知道苦日子终于到头了。。。。。

到了营地，发现传统营地离水源两三米，像我这种习惯帐篷内撒尿，伸手出帐篷倒尿的人还是不要污染水源了。。。

而且最关键的是，传统营地就在一片大平地上，啥挡风的都没有，而我是三季帐。。。。

所以，找了个背风的地方平整了块地，帐篷刚搭起来，就开始下雪，天空大量层云，感觉不妙…

下了十来分钟，雪停了，出帐篷继续搭。。。（塑料布放在上风向挡风。。。

搭完帐篷，就不停的烧水烧水烧水，带的锅儿太小了（其实人家是碗。。。）

冲了点屎将就吃了，还吃了块压缩饼干，喝了1L水，烧了2L热水袋丢睡袋，于是睡觉了

一晚上睡得极好，就是老是发现睡袋被打湿，醒了几次之后，终于想通了，是呼出的水蒸气遇到脸前的睡袋冷凝了。。。

还被雪吵醒几次，听上去挺大的，怕被压垮，结果后来一看没多大，肯定不会垮。。。

9月12日，雪雾，多云，晴

听到下雪不想起床，磨磨唧唧到7点天大量终于起来打了一桶水，满天雪雾，能见度30米，果断回帐篷继续烧水吃早饭。。。

雪落下来之后，把帐篷与挡风用的石头之间的缝隙给封死了。。。难怪帐篷里冷凝水那么多。。。

吃完早饭天气还是那样子，于是躺下玩手机…

直到9点半，雾气才散去，太阳慢慢开始出来了

于是真·起床收拾冲顶包

一边墨迹一边等天气变好，等到10点多11点的时候，天气已经完全转好了

先传一张整体图，绿线为上升路线，黄线为下撤路线

框框和圈圈表示冰裂缝区域

1/4/5尽量不要去惹，深、宽、落差大

2/3可以随便欺负

区域2靠下面的裂缝

区域2较高位置的裂缝，已经逐渐不好惹了

从区域4之间上升的时候过了两个雪桥，夏末初秋都还好，比较稳固

区域4比较浅的裂缝，深的那几个略陡，专心爬着没拍照

过了区域4之后，从右侧上去躲开区域5，基本上就没有什么裂缝了，偶尔有几个很明显的孤立的裂缝，很方便的绕开

在接近顶端的时候出现背隙，不要冲着路绳去，要绕开，有路绳也过不了那个背隙的。。。

上到山脊之后就又变成普通的爬山了。。。

风景还是不错，手机乱按都感觉拿得出手

看向来时的路，画面中央就是刚刚说的“偶尔有几个很明显的孤立的裂缝，很方便的绕开”

山坡北侧的冰裂缝，这才要绕哭（不过走近了看的话，估计有很多都是好欺负的类型。。。

仰望山顶（讨厌的路绳，一路上都要留心着别绊到我腿

半山坡的雪坡，后来发现可以继续走石头，不用爬这雪坡来着，早知道不穿冰爪上来了，磨损好厉害

登顶（图中出现的除了冰镐，全是迪卡侬，简直屌丝之友

啃了半块压缩饼干，闷了半瓶葡萄糖，干了半斤H2O，开始下撤

再借刚刚那张图说明下

绿色是我上来的路，下去的时候，我看着右边那一大片没有冰裂缝又好平摊走着好爽就朝那边走了

走过去之后就来带了之前那幅图的区域5，还真是够酸爽

刚刚那几张照片应该是站在绿色小人处往下拍的

拍完之后继续往下，要过一个雪桥，但是由于裂缝落差太大，冰爪在山脊岩石上磨钝了，无法静态冲击地从高坡裂缝一侧降到雪桥上（红X处），所以保险起见，往回爬了几米绕开了那个裂缝走了旁边一个平缓的雪桥

为啥说无法静态冲击地下降到雪桥呢，因为我已经在那儿滑倒了一次了。。。

还好平时自我确保练习得多(脑内练习。。。)，刚滑倒就停住了，狠狠踢了几下才踢进冰站稳了，没法下了，不敢把生命寄托在运气上

过了这个之后就一路easy了，直到为了躲开区域1的裂缝区，从旁边走结果走到旁边凹盆里的冻土区了，真滑，横切略带上升回到高处又回到正常雪面了。。。

下来之后不到5点，又开始无尽的烧水。。。

其实用碗烧水有个好处，帐篷一直都有火烤，挺暖和的，而且有点事做不至于无聊。。。

9月13日，晴，多云，阴，雪，雪雾

起床就是晴天，真棒！

这么好的天气，赶紧再爬一遍。。。

山谷中稳定的雾气，不抬升，就不会变天

但是高层的卷云预示这未来几天天气不妙

掏出锉刀赶紧锉冰爪，哪知道一根筋上来了，还有就是工匠情怀上来了，一口气锉了一个小时锉得比出厂还锋利。。。。。。。（真蛋疼！

导致9点半才出发。。。

上升路线基本和昨天下撤路线一致，沿着区域3和4之间的无裂缝区域上到区域5右侧，完全绕过区域5不去过雪桥

这是区域3的冰裂缝。。。我都不好意思叫他们冰裂缝了。。。

这是区域4左侧的冰裂缝，高度落差不大，但是有点宽，这雪桥还是不爬为妙

走到区域5右边的时候，感觉那边可以爬的样子，完全不用过裂缝，只是滑坠了后果有点严重，直接掉缝里去了。。。。

后略不表，无言说价值

下撤后才中午12点半，回到营地煮了碗汤，吃了块压缩饼干之后昏昏欲睡

不行，还有这么一大下午呢，于是又收拾东西去旁边那个冰川发育主方向的坡去遛弯

打算联系下各个方向的滑坠制动，哪知道雪不够硬，深度又合适，坡度又不够，就是不穿冰爪，怎么都摔不倒。。。

故意摔倒了也滑不动。。。还得努力向下摔倒。。。。。

好蛋疼。。。。

于是又回到营地继续烧水。。。。

9月14日 雪雾，多云，长时间阵雨

打算今天下撤了，脸晒得痛了，压缩饼干吃不下了，汤料用完了，爬也差不多爬够了

然而，外面在下雪。。。

都8点了，还是这个德行。。。

10点了。。。

实在等不下去了，用手机跟电脑下了几十盘五子棋了。。。

雪稍微停了，出帐篷勘测一下，外带拉屎。。。

还发现了这个，鸽子？山鸡。。。？

拉完屎决定不等了，收包，收帐篷，下撤

结果收一半又开始下雪。。。。像逃难一样。。。

昨天下午烧了一次垃圾，这一夜产生的垃圾也没法空了，连同那个4L的农夫山泉瓶子、塑料布、1/4罐气，都扔营地了。。。

顶着头皮下降到大斜板，雪停了，出了点太阳，真是心情顿时大好

继续下到4600乱石坡，太阳已经完全出来了

沿着前人轨迹找到了那个石缝

然而旁边就有一个很好下的（黄为刚刚那张的石缝，红色为脚踩的点，沿绿色箭头下脚，一步就到底。。。

然而我还是没下这个，而是边下降边找地方过水走的另一侧

地形后略不表，同来时路

走到传统营地（小卖部）附近开始下雨，一会儿停，一会儿下，一会儿停，一会儿下。。。

于是进了牧民的帐篷帐篷借地下包穿衣服

走到两湖之间又开始出太阳，问了一个小妹妹走湖西侧可以出去不，门口那边封了路呢，小妹妹说可以，并指了我一个踩着石头过河的方向

我顺着方向果然看到很多石头，然后开始过河，结果过到一半开始下雨，踩石头上双手撑着登山杖保持平衡，没法穿雨衣。。。。

简直不能更坑了，前一秒还在出太阳，只穿了长袖T恤，然后过完河就快失温了。。。。

过完河之后一路快走稍微恢复点体温，虽然有信号了也不敢停下来看手机和吃东西，水也喝没了，马不停蹄赶到了景区停车场，又饿又渴又冷

权衡之下先解决口渴问题，找了个小卖部买了瓶水，老板人特好，看我太惨了，让我跟武汉来的一个驴友团一起吃了顿饭，算是老板赏的，几天没吃人饭了，而且今天又特别饿，所以这顿饭吃得特别香、特别撑。。。。

吃完之后，老板（女，一个人住，不方便）把我介绍给隔壁大叔住他们家，结果隔壁大叔帮我背包的时候把手机弄掉了，被一个藏娃捡去，要价1000块，我这手机才买成750块。。。。（二手索尼z3

折腾了好久，终于降价到500元，不折腾了，赎回来吧

捡我手机的一家人↓

跟借宿的大叔谈妥230到阿坝（久治中转），于是睡觉

9月15日 少云，晴

起床吃奶茶泡的黄色淀粉糊状物，外加饼子，又吃撑了。。。

吃完坐上大叔家的车去久治

到了久治刚好遇到一个去成都的黑车，说好200去成都，于是给了年保大叔200块车费，换上了去成都的车

到了成都，久治那师傅说是230不是200，卧槽明明我当时问了3遍是不是200，真是。。。算了，懒得纠缠了

下车之后立刻换坐滴滴的顺风车（变相黑车）180元回重庆

到家11点多要到12点，洗澡解包睡觉。。。
